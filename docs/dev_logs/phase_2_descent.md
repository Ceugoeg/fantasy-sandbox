# Phase 2: 意识降临 (Descent of the Mind) - 四层异构集群与代理生态完全体

本阶段的核心目标是将底层沙盒升级为能够承载 LLM 高维意图的四层微服务集群。通过在 C++ 引擎中引入防级联的事件模型与空间索引、在 Node.js 中实现增量通信、在 Python 端搭建带强类型校验的异步工具链，并最终构建带有独立时空控制与潜意识记忆流的“神谕控制台”，彻底打通大模型感知、决策与执行的闭环架构。

## Stage 2.1: 引擎内核重构与事件防波堤 (Core Engine Hardening) [待办]
此阶段纯粹聚焦于 C++ 底层的性能榨取与内存安全，为后续高频的事件风暴与范围判定打下基石。
* **双缓冲事件总线 (Deferred Event Dispatch)**：彻底重构 `EventBus` 的同步分发机制。引入 `pending_events_` 队列，将物理 Tick 期间产生的所有事件压栈，待实体 `update` 循环结束后统一分发。此举将从根源上消除“事件级联 (Event Cascade)”导致的容器迭代器失效与运行时崩溃。
* **O(1) 实体探针 (World Lookup Table)**：在 `World` 核心类中，平行于实体连续内存 `vector` 额外维护一个 `std::unordered_map<int, std::weak_ptr<BaseEntity>>`。使得底层事件回调函数能以 O(1) 的极速验证 `subscriber_id` 的存活性，补全底层生命周期安全的最后一块拼图。
* **空间哈希网格 (Spatial Hash Grid)**：废弃传统 O(N) 的全局实体遍历。利用均匀网格建立 Chunk 索引桶，实现实体视野扫描、AoE 伤害判定等 O(1) 复杂度的空间查询，为未来大模型的雷达探测工具提供极其廉价的算力保障。

## Stage 2.2: 通信管线升级与指令路由 (IPC & Intent Routing) [待办]
此阶段负责解决跨进程管道的带宽灾难，并建立极其严苛的指令准入协议。
* **增量状态同步 (Delta Snapshot)**：在 C++ 实体中引入 `is_dirty` 脏标记。每帧仅向 Node.js 导出坐标或状态发生实质改变的实体 JSON 快照，极大降低 IPC 管道的带宽压力与前端的垃圾回收 (GC) 负担。
* **意图嗅探与路由隔离 (Intent Routing)**：升级 `CommandQueue` 的解析器。当嗅探到输入流为 JSON 格式时，将其打包为专用的 `IntentType::LLM_DIRECTIVE`。主循环捕获后绝不脏手解析，而是直接将其封装为 `LLMCommandEvent` 抛给 `EventBus`，实现上帝仲裁器与实体具体行为逻辑的完美隔离。

## Stage 2.3: 智能大脑与感知代理 (The Python Agent Brain) [待办]
搭建独立于游戏主循环的 Python 异步微服务，作为连接 LLM 与物理世界的桥梁。
* **异步微服务与工具链 (Async Tool Registry)**：基于 FastAPI 等框架建立全异步环境。封装诸如 `scan_surroundings`、`get_entity_status` 等环境感知工具。大模型将通过主动的 Tool Calling 替代被动的全量状态接收，彻底消除 Token 焦虑与协程阻塞。
* **幻觉防火墙 (Hallucination Firewall)**：利用 Pydantic 配合 Hygenar 等结构化解码算法，对大模型输出的 JSON 进行严苛的 Schema 约束与契约拦截。任何越界坐标或非法参数将在 Python 层直接触发重试要求，确保下发至 Node.js 枢纽的指令绝对纯净。
* **世界本体论字典 (World Ontology)**：建立自然语言与 C++ 底层物理 ID 的双向映射中枢。使大模型能以“荒原”、“森林”等高维语义输出环境修改指令，由字典自动翻译为对应的数字掩码阵列后下发至引擎。

## Stage 2.4: 决策神经与状态机演化 (Entity Nervous System & FSM) [待办]
在 C++ 实体的 Policy 层构建能够响应高维指令、具备独立生命周期的智能状态机。
* **优先级状态机 (Priority Interruption FSM)**：打破传统线性状态机，引入意图压栈机制。当实体执行大模型下发的长期宏观意图时，若遭遇底层环境突变（如受到火焰伤害），可立即触发“脊髓反射”切入高优先级的避险状态，危机解除后自动弹栈恢复原有任务。
* **意图保质期与僵尸熔断 (TTL & Timeout Mechanism)**：为大模型下发的任务引入时间戳与最大存活帧数 (Time-To-Live)。若由于寻路受阻等物理原因导致任务超时，状态机必须主动抛弃该意图并向世界广播 `ACTION_FAILED` 事件，防止实体逻辑锁死，并为 Python 层提供重新规划的反馈源。

## Stage 2.5: 渲染解耦与世界法则扩展 (Render Pipeline & Physics) [待办]
解决内容堆积带来的维护痛点，并补全世界的动态物理法则。
* **前端离屏渲染极限优化 (Off-screen Canvas)**：构建与地图等大的隐藏 Canvas 缓存静态地形图层。主循环仅通过单次 `drawImage` 执行视口切片，将 O(W×H) 的重绘开销降为 O(1)，仅在地形被事件真实修改时触发局部刷新。
* **资产管线数据驱动 (Data-Driven Assets)**：将前端硬编码的贴图路径与动画序列帧 (`ANIM_META`) 剥离至独立的 JSON 配置文件，实现零代码修改的新实体热插拔。
* **元胞自动机 (Cellular Automata)**：在 C++ 中引入基于物理 Tick 的地形演化逻辑。处理火焰的相邻蔓延概率、TTL 燃烧倒计时及焦土覆写，并向前端抛出解耦的视觉交互事件（如水面波纹特效）。

## Stage 2.6: 神谕界面与时空操纵 (The God Console & Chronology) [待办]
摒弃纯终端调试，在前端引入极简的“上帝控制台”，实现对引擎时间轴的绝对干预。
* **时空操纵面板 (Chronology Control)**：在前端 UI 增加 `[<] [||] [>] [>>]` 时间控制原语。通过 WebSocket 向 C++ 引擎发送系统指令，动态修改 `TARGET_TPS` 或切换 `is_paused` 标志。实现在不切断网络管线的前提下，完美冻结或快进物理演算。
* **独立输入终端 (The Oracle Terminal)**：提供一个横向交互终端，允许玩家直接输入自然语言指令。输入框聚焦时绝不自动暂停世界，时空控制权完全归属玩家操作，保证极客沙盒的真实流转感。
* **实体探针与系统日志 (Inspector & System Log)**：实现 Canvas 鼠标射线检测，点击实体即可唤出半透明面板，实时监控其 FSM 状态、HP 与当前执行的 LLM 意图；左下角集成滚动日志流，实时反馈 C++ 系统事件与大模型错误反思。

## Stage 2.7: 无感记忆流注入 (Passive Memory Stream) [待办]
在零 LLM Token 消耗的前提下，为世界建立潜意识档案记录。
* **本地向量化监听**：在 Python 微服务中引入轻量级的 ChromaDB/Faiss 数据库。监听 Node.js 转发的高优先级物理事件（如森林火灾、NPC 死亡），在 Python 本地将其拼接为极简陈述句并进行向量化嵌入。
* **潜意识沉淀**：打通从物理世界到记忆空间的单向数据流。所有事件按时间戳静默归档，为未来 Phase 3 中大模型的 Agentic RAG（主动搜索检索）提供充沛的世界历史数据源。

---

## 阶段总结与愿景
当 Phase 2 彻底完结时，异世界沙盒将蜕变为一个真正的**“AI 驱动引擎可视化调试器”**。
玩家可以在网页端实时俯瞰世界的运转；当遇到感兴趣的演化（如森林大火蔓延）时，可一键暂停时空；通过鼠标探针审查一只正处于“逃跑”状态的哥布林，随后在神谕终端中输入指令，看着大模型将其转化为底层的微操意图。在这一切发生的背后，C++ 引擎的高速索引、Node.js 的增量同步与 Python 的异步幻觉防火墙正以极其硬核的四层集群架构静默运转。这不仅是一个游戏 Demo，而是一个为无限复杂逻辑预留了绝对扩展空间的现代数字生命温床。

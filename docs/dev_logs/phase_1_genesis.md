# Phase 1: 创世纪 (Genesis) - 基础设施构建
本阶段的核心任务是“造轮子”，构建一个高内聚、低耦合的可视化沙盘地基，为后续接入复杂游戏逻辑与大模型 Agent 铺平道路。

## Stage 1.1: 物理坐标系与运行时闭环 [已完成]
* 确立 C++ 引擎的内存世界权威，构建基于一维 std::vector 模拟二维网格的高缓存命中率地形矩阵。
* 实现实体基类 `BaseEntity` 及其 Policy 模板注入，构建最基础的 Tick 驱动主循环，并成功跑通终端的 JSON 快照导出。

## Stage 1.2: 进程间通信与异步分发 [已完成]
* 引入 Node.js 承担网关职责，使用 `child_process.spawn` 拉起 C++ 引擎，实现 stdout 流的监听与拦截。
* 搭建 WebSocket 广播管道，解决 C++ 的全缓冲阻塞问题，实现后端运算与前端网页的数据贯通。

## Stage 1.3: 纯数学渲染器与视口隔离 [已完成]
* 建立基于 HTML5 Canvas 的前端 60FPS 渲染循环，独立处理精灵图 (Sprite Sheet) 的序列切割与加载。
* 引入平滑插值 (Lerp) 算法与前端时间戳动画状态机，弥合低频逻辑与高频渲染的视觉断层。
* 建立虚拟摄像机系统，通过手写整数级坐标投影矩阵与视锥剔除 (Camera Culling) 算法，彻底消灭缩放状态下的图块亚像素缝隙黑线。

## Stage 1.4: 纯粹的事件总线架构 (Event Bus) [进行中]
* 目标：在 C++ 中构建发布-订阅模型，彻底解耦实体之间的交互逻辑。
* 待办：利用 `std::any` (或类似类型擦除机制)、`std::function` 及模板特化，实现支持任意类型回调注册的全局消息调度器。
* 意义：这将成为大模型动态编写 Mod 和触发特殊交互的神经枢纽。

## 展望 Phase 2: 法则注入 (Rules of Nature) 
* 将在事件总线完工后启动。涵盖：A* 寻路算法集成、实体碰撞体积抽象、以及基础的生命/战斗属性结算体系构建。
